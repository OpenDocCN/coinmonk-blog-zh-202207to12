# 并非所有代码都是语音

> 原文：<https://medium.com/coinmonks/not-all-code-is-speech-e666bd21c718?source=collection_archive---------37----------------------->

“代码就是语音。”这句话并不新鲜，但自从最近的龙卷风现金制裁以来又重新出现了。对于那些不熟悉情况的人，美国财政部批准了一个名为 Tornado Cash 的应用程序，据称该程序为朝鲜黑客组织清洗窃取的加密资产。Tornado Cash 是以太坊网络上的一个应用程序，允许用户匿名发送资产，尽管所有以太坊交易都是公开的。虽然这不是第一次围绕一个旨在保护公共区块链隐私的应用程序的法律斗争，但这是第一次有人被捕，这是这场骚乱的催化剂。

虽然对制裁和逮捕提出质疑有几个理由，但最突出的理由之一是基于言论自由原则。被捕的龙卷风现金开发者阿列克谢·佩尔采夫(Alexey Pertsev)只写了一些代码，放在以太坊区块链上。代码只是数字和字母，它们必然是语音。第一修正案禁止美国政府剥夺言论自由。所以这将以一个有利于密码开发者的法庭判决而告终，对吗？

这就是我担心的地方。

我希望这是一个能够轻松保护开发者的简单问题，但事实并非如此。“代码即言语”这句话的核心问题是它过度简化了第一修正案的原则。

仅仅因为一个行为涉及到言语的表达或包含字母或数字的东西的传输，并不意味着第一修正案保护该行为不受任何政府干预。相反，重点是如何使用字母/数字，这可能会使代码变得非常复杂，因为代码的使用方式多种多样。

# **当代码明显是语音时**

代码等同于言语的关键例子来自 *Bernstein 诉美国*一案。在这个案例中，原告伯恩斯坦试图公布他开发的一个加密方程式的源代码。当时，美国有法规将这种代码标记为弹药，这意味着伯恩斯坦必须在发布代码之前获得许可，因为发布代码相当于出口弹药。在回答源代码是否有权获得第一修正案保护的问题时，法院表示:

> “我们的结论是，就第一修正案而言，加密软件，以其源代码的形式和被密码学领域的人所使用的形式，必须被视为是表达性的。]"

因此，法院裁定，政府限制伯恩斯坦公布他的加密源代码是非法的。然而，这并不意味着龙卷风现金开发商将有一个灌篮案件对政府。后来在伯恩斯坦的意见中，法院说“我们强调第一修正案的狭隘性。我们并不认为所有的软件都是表达性的。很大一部分肯定不是。”一般来说，当法院处理新技术时，他们“倾向于允许法律在个案的基础上成熟的狭隘观点。”

# **当码不言时**

那么什么时候代码不算语音呢？简短的回答是当代码是一个产品，而不是一个想法。法院在 *Bernstein* 案中表示，“政府的努力旨在阻断科学*思想*的传播，无论是以源代码还是其他方式表达，以区别于加密*产品*，这些努力似乎深入到了第一修正案的核心。”

这种区别的一个很好的例子来自环球影城诉科利案。在这个案例中，被告开发了可以解密 DVD 版权限制的软件，然后允许用户盗版有版权的内容。法院承认，该软件仍然受到第一修正案的一些保护:“一个程序有能力控制计算机的功能，这一事实并不意味着它缺乏传递信息的额外能力，正是信息的传递使第一修正案中的‘言语’成为可能。”但关键问题是:这些信息传达给了谁？如果代码是为了科学进步而与其他程序设计者交流的，它显然是受保护的言论。如果代码只是传达给计算机，那就不再是受第一修正案保护的表达性言论。法院最终认为*环球影城*、*、*中的解密代码严格作为最终用户消费的产品提供，不属于受保护的言论。"这种功能不是第一修正案意义上的言论."

# **这对开发者意味着什么？**

这就是让我担心第一修正案关于区块链技术的发展的模糊之处。我相信法院会发现在类似 GitHub 的论坛上发布 Tornado Cash 应用程序的源代码会受到第一修正案的保护，因为这是一种科学思想的传播。然而，我担心的是，一旦所述源代码被部署在区块链上，那么该代码将被解释为产品，并因此失去其第一修正案的保护。

当比较*伯恩斯坦*和*环球影城*的事实时，部署在区块链上的代码似乎更类似于*环球*提供的完整软件包，而不是在*伯恩斯坦*分发给其他开发者的源代码。在伯恩斯坦案中，原告只是试图发布源代码，以便其他人可以研究它。为了让接收者使用所述加密软件，他们需要在他们自己的硬件上部署它。在 *Universal* 一案中，被告拥有一个应用程序的下载链接，还分享了其他用户可以下载类似软件的网站的超链接。

在 Tornado Cash 的情况下，代码直接存在于区块链上，不需要任何用户在他们自己的硬件上部署软件，它更像是一个完整的软件包。有了区块链上的代码，在软件的发布和使用之间就少了一个步骤。因此，区块链代码至少在某种程度上比发布在 GitHub 上的源代码更接近于一个可用的产品。

# **结论**

尽管如此，这仍然是法院尚未解决的新问题。法院完全有可能认为在 GitHub 和以太坊区块链上发布源代码没有区别。可以肯定地说，区块链是一种非常方便和实用的分发新代码的方式，它可以给其他开发人员提供研究它并做出自己贡献的机会。如果这一论点被接受，很可能区块链法典将得到第一修正案言论的充分保护。不幸的是，在可预见的未来，我们只剩下这种模糊性
